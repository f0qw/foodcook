# FoodCook 家庭菜单管理系统 - 项目总结

## 项目概述

FoodCook 是一个现代化的家庭菜单管理系统，采用前后端分离架构，帮助用户管理家庭菜品、记录用餐体验、管理食材库存。

## 技术架构

### 后端技术栈
- **语言**: Go 1.24+
- **框架**: Gin (Web框架)
- **数据库**: MySQL 8.0+
- **ORM**: GORM
- **缓存**: Redis
- **认证**: JWT
- **日志**: Logrus
- **配置管理**: Viper

### 前端技术栈
- **框架**: Vue 3 (Composition API)
- **构建工具**: Vite
- **UI组件库**: Element Plus
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios
- **样式**: SCSS

## 数据库设计

### 核心表结构
- **users**: 用户表
- **categories**: 菜品分类表
- **dishes**: 菜品表
- **ingredients**: 食材表
- **dish_ingredients**: 菜品食材关联表
- **meal_records**: 用餐记录表
- **meal_record_dishes**: 用餐记录菜品关联表

### 关系设计
- 用户 ↔ 用餐记录 (一对多)
- 分类 ↔ 菜品 (一对多)
- 菜品 ↔ 食材 (多对多)
- 用餐记录 ↔ 菜品 (多对多)

## API设计

### RESTful API
- `GET /api/dishes` - 获取菜品列表
- `POST /api/dishes` - 创建菜品
- `PUT /api/dishes/:id` - 更新菜品
- `DELETE /api/dishes/:id` - 删除菜品
- `GET /api/dishes/search` - 搜索菜品

### 认证API
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/profile` - 获取用户信息

### 用餐记录API
- `GET /api/meal-records` - 获取用餐记录
- `POST /api/meal-records` - 创建用餐记录
- `DELETE /api/meal-records/:id` - 删除用餐记录
